隐私政策

最后更新时间： 2026年6月

Task Flyout（以下简称“本软件”）采用“本地优先”的设计。开发者不运营任何服务器，您的数据保存在您自己的设备上，不会上传给开发者。

1. 各类数据如何存储

  • 邮件：账户配置与邮件元数据（发件人、主题、预览）以 Windows 数据保护接口（DPAPI）加密后缓存在本地；邮件正文按需从您的邮件服务器加载，通常不持久化保存。
  • 日历与任务：同步得到的日程与任务缓存经 DPAPI 加密后保存在本地，用于离线查看。
  • RSS：订阅列表与文章保存在本地的 SQLite 数据库中。部分订阅 URL 或文章可能包含私有内容；删除订阅会从应用数据库移除其记录，但系统备份、文件系统快照或 SQLite 空闲页可能在一段时间内仍保留历史数据。
  • 天气：您选择的城市、坐标与显示设置保存在本地应用设置中。

2. 是否上传服务器

本软件没有自有服务器，不收集任何遥测或使用分析，您的数据不会被收集或上传给开发者。 为实现功能，本软件仅在您的设备与下列您主动配置的官方/第三方服务之间直接通信：

  • Google API、Microsoft Graph API（日历、任务、邮件）；
  • 您配置的 IMAP/SMTP 邮件服务器；
  • 您订阅的 RSS 源所在站点；
  • 天气服务（Open-Meteo 或 wttr.in）：获取天气时会向其发送您选择的地点信息。

3. OAuth 令牌与密码如何保护

  • OAuth 访问/刷新令牌：通过 Windows DPAPI（CurrentUser 范围并附加额外 entropy）加密保存在本地。加密密钥与您当前的 Windows 用户账户绑定，其他用户或其他设备无法解密。
  • Microsoft 登录：采用官方 MSAL 库，令牌缓存同样以 DPAPI 加密保存在本地。
  • IMAP/SMTP 密码：保存在 Windows 凭据管理器（Credential Manager） 中，而非明文文件。
  • 最小权限：仅申请实现功能所必需的最少 OAuth 权限范围，并遵守 Google API 服务用户数据政策。

4. 删除数据

在应用内移除某个账户会删除应用记录、IMAP/SMTP 密码、Google 授权材料和对应缓存。Microsoft 登录使用 Windows 加密的共享平台令牌缓存；应用会移除认证记录，但缓存条目可能保留到其自然过期或由 Microsoft 帐户/Windows 管理。卸载会移除应用包数据，但系统备份、通知历史和文件系统快照不受本软件控制。

5. 联系我们

如果您对本隐私政策有任何疑问或安全担忧，请通过 GitHub Issues 联系开发者。

Privacy Policy

Last Updated: June 2026

Task Flyout ("the App") is local-first by design. The developer operates no servers; your data is kept on your own device and is never uploaded to the developer.

1. How each type of data is stored

  • Mail: Account configuration and message metadata (sender, subject, preview) are cached locally, encrypted with the Windows Data Protection API (DPAPI). Full message bodies are loaded on demand from your mail server and are generally not persisted.
  • Calendar & tasks: Synced events and tasks are cached locally, encrypted with DPAPI, for offline viewing.
  • RSS: Your subscriptions and articles are stored in a local SQLite database. Feed URLs and articles can contain private content; deleting a feed removes its records from the app database, but system backups, filesystem snapshots, or SQLite free pages can retain historic data for a period of time.
  • Weather: Your chosen city, coordinates, and display preferences are stored in local app settings.

2. Is anything uploaded to a server?

The App has no backend of its own, collects no telemetry or usage analytics, and never sends your data to the developer. To provide its features it communicates only between your device and the following official / third-party services that you configure:

  • Google API and Microsoft Graph API (calendar, tasks, mail);
  • The IMAP/SMTP mail server you configure;
  • The websites hosting the RSS feeds you subscribe to;
  • The weather service (Open-Meteo or wttr.in): your chosen location is sent to it when fetching the forecast.

3. How OAuth tokens and passwords are protected

  • OAuth access/refresh tokens: Encrypted at rest with the Windows DPAPI (CurrentUser scope plus additional entropy). The encryption key is bound to your current Windows user account, so other users or other machines cannot decrypt it.
  • Microsoft sign-in: Uses the official MSAL library; its token cache is likewise stored locally, encrypted with DPAPI.
  • IMAP/SMTP passwords: Stored in the Windows Credential Manager, not in a plaintext file.
  • Least privilege: The App requests only the minimum OAuth scopes needed for its features and complies with the Google API Services User Data Policy.

4. Deleting your data

Removing an account deletes the app record, IMAP/SMTP password, Google authorization material, and associated cache. Microsoft sign-in uses a Windows-encrypted shared platform token cache; the App removes its authentication record, but cache entries can remain until they expire naturally or are managed through the Microsoft account or Windows. Uninstalling removes app package data, but system backups, notification history, and filesystem snapshots are outside the App's control.

5. Contact Us

If you have any questions or security concerns regarding this Privacy Policy, please contact the developer via GitHub Issues.
